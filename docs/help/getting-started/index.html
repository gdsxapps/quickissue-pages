QuickIssue Setup

Step 1: Create a GitHub Token

Click the "Start Initial Setup" button in QuickIssue, then click "Issue Token on GitHub" to open the GitHub token creation page.

GitHub Authentication

GitHub requires authentication before you can create a token. There are two methods available:

  • Email verification - A code will be sent to your registered email
  • GitHub Mobile app - Approve the request from your phone

Option A: Email Verification

Click "Verify via email" and enter the code sent to your email.

Enter the code you received via email here.

Option B: GitHub Mobile App

If you have the GitHub Mobile app installed, click "Use GitHub Mobile" and enter the displayed number in your app.

Please enter this number in the GitHub mobile app.

Once authenticated, you will be redirected to this screen.

Configure Your Token

Once authenticated, you'll see the token creation screen. Configure the following:

  1. Token name - Enter a descriptive name like "QuickIssue"
    (Names that have been used before cannot be reused)
  2. Expiration - Choose how long the token will be valid (recommended: 30‚Äì90 days)

About Token Expiration

For security reasons, we recommend setting an expiration period of 30 to 90 days rather than using a long-lived or non-expiring token. While longer expirations are more convenient, shorter periods reduce risk if your token is ever compromised.

üí° Tip: When your token expires, simply follow the same steps to create a new token and update it in QuickIssue. The app will prompt you when re-authentication is needed.

Understanding the repo Scope

QuickIssue requires the repo scope to create issues on your behalf. However, you should understand what this permission grants:

  • The repo scope provides full read and write access to all your public and private repositories
  • This includes access to code, commit statuses, repository invitations, collaborators, and more
  • QuickIssue only uses this permission to create issues, but the token itself has broader capabilities

üìå For public repositories only: If you only need to create issues in public repositories, you can select the more limited public_repo scope instead. This restricts access to public repositories only.

Security Best Practices

‚ö†Ô∏è Treat your token like a password. Never commit it to a repository, share it with others, or post it publicly. If you accidentally expose your token (e.g., in a public repository), immediately revoke it on GitHub and generate a new one.

Advanced: Fine-grained Personal Access Tokens

For users who want more granular control over permissions, GitHub offers Fine-grained Personal Access Tokens. These allow you to limit access to specific repositories and select only the exact permissions needed. For more information, see the GitHub Docs on Fine-grained tokens.

Scroll down and click the "Generate token" button.

Copy Your Token

Your new token will be displayed. Click the copy button to copy it to your clipboard.

‚ö†Ô∏è Warning: The token is only displayed once! Make sure to copy it now. If you lose it, you'll need to create a new token.

Step 2: Paste Token in QuickIssue

Return to QuickIssue. In the "Paste your token" section:

  1. Paste the token you copied from GitHub
  2. Click "Save Token" to save it

Step 3: Add Destination Repository

Now configure where your issues will be sent. Click "Add destination" to select a repository.

Press Add destination to select a destination.

A list of available repositories will be displayed. Click the + button next to the repository you want to use as a destination.

üí° Note: In the free version, you can register only one repository. Upgrade to Pro for multiple destinations.
